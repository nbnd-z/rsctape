# rsc-tape Guide

A practical walkthrough for using rsc-tape in your project.

## Prerequisites

- Node.js 18+
- An RSC-compatible framework (Next.js 13+, Waku, Parcel, or custom)
- MSW 2.x in your test setup

## Step 1: Install

```bash
npm install rsc-tape --save-dev
```

## Step 2: Initialize

```bash
npx rsctape init
```

This does three things:
1. Detects your framework (Next.js, Waku, or unknown)
2. Creates `rsctape.config.json` with defaults
3. Generates a framework-specific entry point file

If the entry point file already exists, it prints the code snippet for you to add manually.

## Step 3: Enable interception

### Next.js

`rsctape init` creates `instrumentation.ts`:

```typescript
export async function register() {
  if (process.env.NODE_ENV === 'development') {
    const { register } = await import('rsc-tape');
    register();
  }
}
```

Next.js automatically calls this file's `register()` on server startup.

### Waku / Custom server

Add to your server entry point, before `http.createServer`:

```javascript
import { register } from 'rsc-tape';

if (process.env.NODE_ENV === 'development') {
  register();
}
```

`register()` must be called before any `http.createServer()` call — it patches the function itself.

## Step 4: Capture fixtures

Start your dev server normally:

```bash
npm run dev
```

You should see:
```
[rsc-tape] Recording server actions...
[rsc-tape] Detected framework: next
```

Now use your app. Every Server Action call is automatically captured to `./fixtures/actions/` (or your configured directory).

### Verbose mode

Set `RSCTAPE_VERBOSE=true` to see each capture:

```bash
RSCTAPE_VERBOSE=true npm run dev
```

```
[rsc-tape] Captured action: abc123def [200]
[rsc-tape] Captured action: xyz789ghi [200]
```

## Step 5: Inspect fixtures

```bash
npx rsctape list
```

## Step 6: Generate MSW handlers

### One-time generation

```bash
npx rsctape mock -o ./src/mocks/server-action-handlers.ts
```

### Watch mode

Keep handlers in sync as you capture new fixtures:

```bash
npx rsctape mock -o ./src/mocks/server-action-handlers.ts --watch
```

### Filter specific actions

```bash
npx rsctape mock -o handlers.ts --actions abc123 xyz789
```

## Step 7: Use in tests

```typescript
// src/mocks/handlers.ts (generated by rsctape mock)
import { handlers } from './server-action-handlers';

// In your MSW setup
import { setupServer } from 'msw/node';

const server = setupServer(...handlers);

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());
```

Your tests now replay real Server Action responses.

## Step 8: Generate types (optional)

```bash
# TypeScript interfaces
npx rsctape types

# JSDoc typedefs
npx rsctape types --jsdoc

# Single action
npx rsctape types abc123def
```

For form submissions, you get an interface:
```typescript
export interface Abc123defInput {
  username: string;
  age: number;
}
```

For programmatic calls (`action(arg1, arg2)`), you get a tuple:
```typescript
export type Abc123defInput = [string, number];
```

## Comparing fixtures

When you change your Server Action code, HMR generates a new action ID. The old fixture stays on disk. Compare them:

```bash
# Quick comparison (input structure + output hash)
npx rsctape diff oldActionId newActionId

# Full comparison (includes RSC Payload line diff)
npx rsctape diff oldActionId newActionId --full
```

## Cleaning up

```bash
# Delete a single fixture
npx rsctape delete abc123def

# Delete all fixtures (manual)
rm -rf ./fixtures/actions/
```

## Configuration reference

`rsctape.config.json`:

```json
{
  "fixtureDir": "./fixtures/actions",
  "ignore": ["**/internal-*", "debug-*"]
}
```

- `fixtureDir` — directory for fixture files. Relative to project root.
- `ignore` — glob patterns. Actions matching any pattern are not captured.

## Ignoring fixtures in git

Add to `.gitignore`:

```
fixtures/actions/
```

Or commit them if you want fixtures as part of your test suite.

## Troubleshooting

### "No fixtures found" after running dev server

- Check that `register()` is called before `http.createServer()`
- Verify `NODE_ENV=development` is set
- Try `RSCTAPE_ENABLED=true` to force enable
- Check `rsctape.config.json` — is `fixtureDir` correct?

### Fixtures are empty or have `parseFailed: true`

- The request might not be `multipart/form-data` (some frameworks use different encodings)
- Check the `.meta.json` file for details

### MSW handlers don't match

- Server Action handlers use `http.post('*')` with `Next-Action` header matching
- Make sure your MSW setup doesn't have conflicting handlers that match first

### Action IDs keep changing

This is normal during development — HMR/recompile generates new IDs. Old fixtures stay on disk. Use `rsctape diff` to compare old and new.

## Programmatic usage

```typescript
import {
  register,
  createHandler,
  generateHandlers,
  loadConfig,
  detectFramework,
} from 'rsc-tape';

// Enable interception
register({ fixtureDir: './my-fixtures', verbose: true });

// Generate handler code for a single fixture
const code = createHandler('actionId', {
  input: { username: 'alice' },
  output: '0:{"result":"ok"}\n',
});

// Generate all handlers to a file
await generateHandlers({
  fixtureDir: './fixtures/actions',
  outputPath: './handlers.ts',
  actionIds: ['abc123'], // optional filter
});

// Detect framework
const framework = await detectFramework(); // 'next' | 'waku' | 'parcel' | 'unknown'
```
